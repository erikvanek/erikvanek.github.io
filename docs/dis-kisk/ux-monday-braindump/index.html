Success metric for the talk: At least some people leave thinking "okay, maybe AI actually changes how I work" — not in a hype way, but in a "I could spend more time on high-leverage human stuff" way. Better conversations, better relationships, less busywork.

The workflow, roughly:

  1. Do the research properly — plan, script, genuine conversations.
  2. Process the data (more on this later — you'll go deeper).
  3. Store everything as markdown files in a shared repository (not behind a paid tool like Dovetail/Condens). Plain text that anyone can read.
  4. Anyone on the team can point their AI tool of choice at that repo, get up to speed, process data, figure out what's next — next interview, next alignment meeting, whatever.

Key tension you're naming: Tools like Condens and Dovetail are good, you even pay for Dovetail personally. But for team contexts (especially cost-constrained ones), markdown in a shared repo is more democratic, more accessible, and honestly more efficient now that AI can parse it.

Implicit argument: The file format and accessibility matter more than the tool. If your research artifacts are structured plaintext in a place everyone can reach, AI closes the gap that expensive research platforms used to fill.
