## 1. The core thesis — single narrative, project pages as appendix

From Ridd's *[Is it time to rethink your portfolio structure?](https://www.dive.club/ideas/is-it-time-to-rethink-your-portfolio-structure)*:

> "Project pages fragment your story. Putting all of your best work on one page allows you to present a **single compelling narrative** about yourself."

The move: make the homepage the hero (a "sizzle reel" of the strongest work in one place), and **demote project pages to an appendix** — supplementary, not the primary way people consume the work.

**Why this matters here:** it is *exactly* the `plan.md` decision — one journey walkthrough (`2023 → 2026` along the maturity curve) as the primary artifact, with the legacy `/dis-kisk/` pages as opt-in depth links. Erik arrived at it independently; Ridd is the canonical framing. **Working phrase: the walkthrough is the narrative; the legacy site is the appendix.**

## 2. The four tactics — each tied to a dis-kisk move

| Ridd's tactic | Source | dis-kisk application |
|---|---|---|
| **Hook, not a label. Create a curiosity gap** — title projects like a tweet, not a category | [Showcasing work](https://www.dive.club/ideas/showcasing-work) | The hero title (D10) + per-territory collage chips. "Naučil jsem se řemeslo. Teď ho učím." is already a hook. Run the same test on every territory caption. |
| **Recipe-video structure** — lead with the gorgeous finished shot, *then* reveal process; never open on sticky notes | [Showcasing work](https://www.dive.club/ideas/showcasing-work) | Each collage territory leads with the strongest *outcome* (Adam won the directorship; the MoI medal; ministry secretaries in the room), not methodology. Process lives on the depth link. |
| **Bite-sized stories > one long case study** — "more shots on goal" | [Showcasing work](https://www.dive.club/ideas/showcasing-work) | The six-territory structure already does this. Six small earned stories read as a bigger body of work than one monolith. |
| **Video-first / start with the presentation, not the website** — record yourself talking through decisions; "they want to know what it *feels like* to work with you" | [Why I went video-first](https://www.dive.club/ideas/why-i-went-video-first-with-my-design-portfolio) | Same instinct as "the site IS the exam presentation." Post-exam: short Screen Studio walkthroughs are the #1 commercial upgrade — the bike/dictation/LLM workflow ending is *tailor-made* for a video of Erik actually working that way. |

Throughline of all four (and the same claim as `Storytelling.md`): **"Your work is only as good as the story you can tell about it."**

## 4. The three saved references — one lesson each

- **[Rik Williams](https://rikwilliams.net/)** — *curation as a signal of expertise.* No project grid; six curated featured posts, a "What people say" testimonial strip, a footnoted personality flourish. **Selectivity reads as seniority** — the same instinct as the opt-in/bottom-up principle.
- **[Mehdi Bouakkadia](https://mehdib.me/projects)** — *outcome-first blurbs, never aesthetic-first.* Every blurb is a transformation or a number ("$90K → $700K/month") with the exact role named. dis-kisk equivalent: lead each beat with the concrete third-party outcome (won the directorship, MoI medal, CEO callback years later).
- **[Zach Klein — Specialization is for insects](https://substack.zachklein.com/p/specialization-is-for-insects)** — *"I have permission to be a prism."* The generalist as a strength, not an apology. The philosophical spine under the six-role Lego figure: the costumes aren't scattered, they're a prism. The rebuttal if anyone in the exam room reads "teaches + designs + does public-services" as unfocused.

## 5. How to use this along the way (working checklist)

Wired into `plan.md` at the points where each applies — pull it up at these moments:

1. **When writing/reviewing any stage or territory caption (Phase 1, Phase 2, D10):** run the **"hook, not a label"** test. If the caption names a category instead of opening a curiosity gap, rewrite it.
2. **When ordering content inside a territory/beat:** **recipe-video order** — strongest outcome first, process on the depth link. Never lead with sticky notes.
3. **When tempted to deepen the legacy site:** remember **the legacy site is the appendix.** Energy goes to the single-narrative walkthrough; project pages are opt-in depth, not the main event.
4. **When the closing/Beat 4 needs proof:** the workflow claim is the most *video-able* asset in the portfolio. Park video-first as the #1 post-exam commercial upgrade (see `plan.md` §6).
5. **If "generalist = unfocused" comes up (Q&A prep):** the prism framing (Zach Klein) is the rebuttal — range presented as coherence (Catt Small is the working example).
